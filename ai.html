Execution Rules

protect momentum
Rule 1

Never run duplicates in parallel

One prompting course, one MCP course, one agent course, one Claude course. Everything else becomes archive or reference.

Rule 2

Finish before you expand

Do not add new AI courses until you have finished a module checkpoint and built a concrete artifact from it.

Rule 3

Build every month

Each phase should end with something you can demo: prompt set, RAG prototype, MCP workflow, or agentic assistant.

Rule 4

Read alongside implementation

Books are not a separate lane. Use them to sharpen your judgment while the courses give you execution reps.

What I recommend you remove from the live page

declutter

Remove duplicated Claude beginner tracks, repeated MCP intros, and low-signal challenge content. Keep them in your file archive if you want, but do not keep them in the active roadmap. The active roadmap should feel smaller, cleaner, and more dangerous.
